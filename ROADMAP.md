# oh-my-somnia 활용 로드맵 🗺️

> 목표: **많은 사람들을 도와줄 수 있는 시스템.**
> oh-my-somnia는 실행할수록 똑똑해지는 하네스이므로, 반복 작업이 있는
> 프로젝트에 붙일수록 학습(게놈)이 복리로 쌓인다.

## 적합성 원칙 — 어디에 쓰면 빛나는가

| 조건 | 이유 |
|---|---|
| ① 객관적 fitness가 있다 | 테스트/빌드/검증 스크립트의 종료 코드가 pass/fail을 판정해야 진화(SELECT)가 정확함 |
| ② 비슷한 작업이 반복된다 | 게놈 학습이 다음 작업에 재사용되어 복리 효과 |
| ③ 샌드박스에서 돌릴 수 있다 | 실패한 시도가 실제 프로젝트를 오염시키지 않음 |

주관적 기준("예쁘게", "감동적으로")만 있는 일은 AI 저지에만 의존하게 되어 약함.
그런 일은 fitness를 **검증 스크립트로 번역**한 뒤에 맡길 것.

## 단계별 로드맵

### Phase 1 — 오늘: 셀프호스팅 루프 완성 🥇

somnia가 **자기 자신의 pytest 테스트 슈트를 작성**하게 하고, 성공하면
`.somnia/config.toml`에 `fitness_command = "python -m pytest -q"`를 고정.

- 이후 이 레포의 모든 개선 작업(`somnia run "worktree 서브디렉터리 지원 추가"` 등)이
  자동으로 테스트 검증을 통과해야만 머지됨 — 다윈적 자기개선 루프의 완성.
- 검증: 머지된 테스트를 사람이 직접 실행·리뷰하고 커밋.

### Phase 2 — 이번 주: 학생 과제 자동 채점 하네스 (파급력 최대)

과제별 루브릭을 pytest로 작성한 채점기 레포. 새 과제가 생기면
`somnia run "X 과제 채점기 추가"`로 확장.

- fitness = 모범답안·오답 샘플에 대한 채점기 자체 테스트.
- 강사는 채점 시간을 돌려받고, 학생은 즉시 피드백을 받음.
- 시작은 과제 1개짜리 최소 채점기로 — 작게 시작해서 진화시키기.

### Phase 3 — 다음: 둘 중 하나 선택

**3a. 이스포츠 경기 데이터 분석 CLI (학생 코칭용)**
- 경기 JSON(픽스처 데이터로 시작 → 이후 Riot API)에서 픽밴 경향·성장 곡선·
  오브젝트 타이밍 리포트 생성.
- fitness = 픽스처 대비 pytest. 분석 지표를 하나씩 `somnia run`으로 추가.
- 게놈에 "결측 필드 처리" 같은 데이터 도메인 휴리스틱이 쌓임.

**3b. GDD 밸런스 시뮬레이터 (/gdd 스킬의 실행 엔진화)**
- 캐릭터/아이템 스탯 → 승률 분포 몬테카를로 시뮬레이션 라이브러리.
- fitness = property-based 테스트("어떤 스탯 조합에서도 승률이 0/1로
  붕괴하지 않는다" 등).
- 이스포츠 최적화 게임 기획 전문성과 직결.

### Phase 4 — 보조: 교안 QA 파이프라인

교안(마크다운/슬라이드) 레포에 검증 스크립트를 두고 fitness로 사용:
깨진 링크, 용어 일관성, 퀴즈 정답 키 누락, 슬라이드당 글자수 상한.
critique-deck(내용 비평)과 상호보완 — 이쪽은 구조 QA 자동화.

### Phase 5 — 장기 비전: 커뮤니티 게놈 허브 🌍

somnia의 진짜 자산은 진화된 유전자(genome). 유전자 md 파일을 모아 공유하는
공개 레포를 만들고, 스키마 검증 스크립트를 fitness로 유지.

- **한 사람의 실패에서 배운 교훈이 모두의 하네스를 똑똑하게 만든다** —
  "많은 사람들을 도와주는 시스템" 미션 그 자체.
- README의 "학습(게놈)도 머신 간에 공유하려면" 섹션의 확장판.

## 운영 수칙

- 비용 통제: `.somnia/config.toml`의 `max_budget_usd`, `generations`으로 상한 설정.
- 새 프로젝트에 붙일 때: `somnia init` → `fitness_command` 설정이 첫 단추.
- 게놈 관리: `somnia status`로 승률 추적, 이상한 유전자는 `somnia genome rm`,
  주기적으로 `somnia evolve`로 히스토리에서 후보 채굴.
- 머신 간 동기화: 코드(이 레포)는 push/pull, 학습은 README 멀티 머신 섹션 참고.
